2.2 DDL Lengkap:
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    nama_lengkap VARCHAR(255) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role user_role DEFAULT 'user',
    is_verified BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE kategori_transaksi (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL,
    nama_kategori VARCHAR(100) NOT NULL,
    tipe tipe_transaksi NOT NULL,
    CONSTRAINT fk_kategori_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE TABLE transaksi (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL,
    kategori_id UUID NOT NULL,
    tanggal DATE NOT NULL,
    nominal DECIMAL(15, 2) NOT NULL CHECK (nominal > 0),
    keterangan TEXT,
    tipe tipe_transaksi NOT NULL,
    CONSTRAINT fk_transaksi_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    CONSTRAINT fk_transaksi_kategori FOREIGN KEY (kategori_id) REFERENCES kategori_transaksi(id) ON DELETE RESTRICT
);

BAB V DEPLOYMENT DAN HOSTING
5.1 Metode Hosting: VPS / Cloud
5.2 URL Deployment: https://bukuku-232546922979.asia-southeast1.run.app

5.3 Langkah Deployment:
Aplikasi di-deploy ke Google Cloud Run dengan langkah berikut:
1. Build dan push Docker image ke Google Artifact Registry untuk backend dan frontend.
2. Jalankan perintah deployment menggunakan gcloud CLI:
   - Backend:
     gcloud run deploy bukuku-backend \
       --image asia-southeast2-docker.pkg.dev/[PROJECT_ID]/repo/bukuku-backend:latest \
       --region asia-southeast2 --allow-unauthenticated --min-instances 1
   - Frontend:
     gcloud run deploy bukuku-frontend \
       --image asia-southeast2-docker.pkg.dev/[PROJECT_ID]/repo/bukuku-frontend:latest \
       --region asia-southeast2 --allow-unauthenticated --min-instances 1

5.4 Perubahan Konfigurasi (Lokal ke Hosting):
Lokal: Database menggunakan localhost (PostgreSQL), API Base URL http://localhost:5000, semua konfigurasi di file .env lokal.
Hosting: Database menggunakan Cloud SQL (PostgreSQL), URL backend mengarah ke service Cloud Run, konfigurasi melalui Secret Manager (Environment Variables).
